# Checklists

Action-oriented checklists for recurring tasks. ADRs explain *what* we decided and *why*; checklists tell you *what to verify* on a given task so nothing falls through the cracks.

## Index

- [new-project-checklist.md](new-project-checklist.md) — bootstrapping or auditing a new Rising Company app

## How to use

Walk top to bottom. Each item is a binary check — done or not done. If something doesn't apply, say so explicitly rather than skipping silently. If you find yourself wanting to skip an item, that's usually a sign to slow down, not speed up.

If a checklist is wrong or incomplete for what you actually do, fix the checklist in the same change.
